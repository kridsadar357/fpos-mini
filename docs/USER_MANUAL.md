## 1. ภาพรวมแอป

FUEL POS เป็นระบบขายหน้าปั๊มน้ำมัน ใช้งานบนแท็บเล็ตหรือมือถือ รองรับ:

- ขายน้ำมันตาม **จำนวนเงิน (บาท)** หรือ **จำนวนลิตร**
- หลายช่องทางชำระเงิน (เงินสด / โอน / QR / บัตร)
- เปิด–ปิดกะ พร้อมสรุปยอด
- พิมพ์ใบเสร็จผ่าน Bluetooth
- ตั้งค่าปั๊ม ราคาน้ำมัน โปรโมชั่น คลัง ฯลฯ (ตาม Package License)

![หน้า Splash / โลโก้แอป](manual/images/01-splash.png)

*รูปที่ 1 — หน้าเปิดแอป (Splash)*

## 3. เข้าสู่ระบบและเปิดกะ

### 3.1 เข้าสู่ระบบ (Login)

1. เปิดแอป FUEL POS
2. กรอก **ชื่อผู้ใช้** และ **รหัสผ่าน**
3. กด **เข้าสู่ระบบ**

ค่าเริ่มต้นสำหรับทดสอบ (เปลี่ยนใน production):

| บทบาท | ชื่อผู้ใช้ | รหัสผ่าน |
|--------|-----------|----------|
| Admin | `admin` | `admin123` |

![หน้า Login](manual/images/04-login.png)

*รูปที่ 4 — หน้าเข้าสู่ระบบ*

- ติ๊ก **จำชื่อผู้ใช้** ได้หากใช้เครื่องประจำ
- ระบบมีเสียง TTS แจ้งเมื่อเปิดหน้า login

### 3.2 เปิดกะขาย

หลัง login สำเร็จ หากยังไม่มีกะเปิดอยู่ ระบบจะถาม **เปิดกะขาย**

1. กรอก **เงินสดในลิ้นชักเริ่มต้น** (บาท) — ใส่ `0` ได้หากไม่ต้องการบันทึก
2. กด **เปิดกะ**

![Dialog เปิดกะ](manual/images/05-open-shift.png)

*รูปที่ 5 — เปิดกะขาย*

> **สำคัญ:** ต้องมีกะเปิดอยู่ก่อนทำรายการขาย แต่ละพนักงานเปิดกะของตนเอง

| ออกจากระบบ | Logout |

## 5. ขายน้ำมัน

### 5.1 เลือกตู้จ่ายและมือจ่าย

1. แตะ **ตู้จ่าย** (เช่น ตู้จ่าย 1)
2. แตะ **มือจ่าย / หัวจ่าย** (เช่น B7 หัว 1)
3. ระบบแสดงประเภทน้ำมันและราคาต่อลิตร

![เลือกตู้จ่ายและมือจ่าย](manual/images/07-select-nozzle.png)

*รูปที่ 7 — เลือกตู้จ่ายและมือจ่าย*

### 5.2 กรอกจำนวน

สลับโหมดได้ 2 แบบ:

| โหมด | วิธีใช้ |
|------|---------|
| **จำนวน (บาท)** | กรอกยอดเงินที่ต้องการขาย |
| **จำนวน (ลิตร)** | กรอกปริมาณลิตร |

- ใช้ **แป้นตัวเลข** ด้านขวา หรือปุ่มลัด **100 / 200 / 300 / 500 / 1k**
- กด **ล้าง** เพื่อเริ่มใหม่

![กรอกจำนวนบาท/ลิตร](manual/images/08-enter-amount.png)

*รูปที่ 8 — กรอกจำนวนขาย*

> **การปัดเงิน:** เมื่อกรอกเป็น **ลิตร** ระบบจะคำนวณยอดเงินแล้ว **ปัดขึ้นเป็นบาทเต็ม** (ไม่มีสตางค์) เช่น 10 ลิตร × 31.94 บ./ล. → **320 บาท**

### 5.3 เลือกลูกค้า (ไม่บังคับ)

- แตะ **เลือกลูกค้า / เพิ่มลูกค้า** เพื่อผูกลูกค้า ทะเบียนรถ หรือข้อมูลใบกำกับภาษี
- แตะ **บัตรฟลีท** ที่แป้นตัวเลข สำหรับลูกค้าฟลีทการ์ด

![เลือกลูกค้า](manual/images/09-select-customer.png)

*รูปที่ 9 — เลือกลูกค้า (ถ้ามี)*

### 5.4 ชำระเงิน

1. กดปุ่ม **ชำระเงิน** (สีเขียว)
2. เลือก **ช่องทางชำระเงิน**
3. ดำเนินการตามช่องทาง (ดูหัวข้อถัดไป)

![เลือกช่องทางชำระเงิน](manual/images/10-payment-method.png)

*รูปที่ 10 — เลือกช่องทางชำระเงิน*

### 5.5 ยกเลิก / พักบิล / พิมพ์

| ปุ่ม | การทำงาน |
|------|----------|
| **ยกเลิก** | ล้างรายการปัจจุบัน |
| **พักบิล** | เก็บรายการไว้ทำต่อภายหลัง |
| **พิมพ์** | พิมพ์ใบแจ้งหนี้/ใบเสร็จร่าง (ก่อนชำระ) |

## 6. ชำระเงินและใบเสร็จ

### 6.1 เงินสด

1. หลังเลือก **เงินสด** ระบบเปิดหน้า **รับเงิน**
2. กรอก **จำนวนเงินที่รับ** (หรือใช้ปุ่มลัด)
3. ตรวจสอบ **เงินทอน**
4. กดยืนยัน → ไปหน้า **สรุปรายการ**

![หน้ารับเงินสด](manual/images/11-receive-cash.png)

*รูปที่ 11 — กรอกเงินที่รับ (เงินสด)*

### 6.2 โอน / QR / บัตรเครดิต

- ไม่ต้องกรอกเงินทอน
- ระบบไปหน้า **สรุปรายการ** ทันที

### 6.3 สรุปรายการและยืนยัน

หน้า **สรุปรายการ** แสดง:

- ประเภทน้ำมัน / ลิตร / ราคา
- โปรโมชั่น (ถ้ามี — Package Pro ขึ้นไป)
- ยอดรวม (บาทเต็ม)
- เงินรับ / เงินทอน (กรณีเงินสด)

1. ตรวจสอบยอด
2. กด **ยืนยัน**
3. เลือก **พิมพ์ใบเสร็จ** หรือ **ไม่พิมพ์**

![หน้าสรุปรายการ](manual/images/12-summary.png)

*รูปที่ 12 — สรุปก่อนยืนยันขาย*

> การขายจะบันทึกพร้อมตัดสต็อกถังอัตโนมัติ หากสต็อกไม่พอระบบจะแจ้งเตือนและไม่บันทึกรายการ

### 6.4 หน้าสำเร็จและพิมพ์ซ้ำ

หลังขายสำเร็จ:

- แสดงเลขที่ใบเสร็จและรายละเอียด
- **ลองพิมพ์อีกครั้ง** — หากครั้งแรกพิมพ์ไม่สำเร็จ (ระบบลองพิมพ์อัตโนมัติ 3 ครั้ง)
- **เริ่มรายการใหม่** — กลับหน้าขาย

![หน้าขายสำเร็จ](manual/images/13-success.png)

*รูปที่ 13 — หน้าหลังขายสำเร็จ*

## 7. พักบิล / เรียกคืนรายการ

### พักบิล

1. กรอกรายการขายให้ครบ (เลือกน้ำมัน + จำนวน)
2. กด **พักบิล**

### เรียกคืน

- แตะ **พักไว้** ที่ Sidebar หรือแป้นตัวเลข
- เลือกรายการที่ต้องการ → ระบบโหลดกลับมาทำต่อ

![รายการพักไว้](manual/images/14-suspended.png)

*รูปที่ 14 — รายการที่พักไว้*

## 8. ปิดกะขาย

เมื่อจบกะหรือส่งมอบกะ:

1. กดปุ่ม **ปิดกะ #xxx** ที่แถบบนของหน้า POS
2. ตรวจสอบ **สรุปยอดกะ**:
   - จำนวนรายการ / ยอดขายรวม
   - แยกน้ำมัน / สินค้า
   - แยกช่องทางชำระ (เงินสด, โอน, QR, บัตร)
   - เงินสดเริ่มต้น + ยอดขายเงินสด → **เงินในลิ้นชัก (คาดการณ์)**
3. กรอก **เงินสดจริงในลิ้นชัก** ตอนปิดกะ
4. กด **ยืนยันปิดกะ**
5. เลือก **เปิดกะใหม่** หรือ **ออกจากระบบ**

![สรุปก่อนปิดกะ](manual/images/15-close-shift.png)

*รูปที่ 15 — สรุปยอดก่อนปิดกะ*

> หากมีรายการขายค้างอยู่ (ยังไม่ชำระ) ระบบจะถามว่าต้องการล้างและปิดกะหรือไม่

## 11. ระบบหลังบ้าน (Backend)

เข้าได้จาก Sidebar → **ตั้งค่า** (เฉพาะ **Admin**)

![เมนูระบบหลังบ้าน](manual/images/19-backend-home.png)

*รูปที่ 19 — หน้าเมนูระบบหลังบ้าน*

### รายการเมนู

| เมนู | หน้าที่ | Package |
|------|---------|---------|
| **นำเข้าน้ำมัน** | บันทึกรับน้ำมันเข้าถัง | Pro, Enterprise |
| **คลังน้ำมัน** | ดูสต็อก / กราฟ / ปรับยอดถัง | Standard (manual), Pro+ (เต็ม) |
| **ภาพรวมรายวัน** | รายงานยอดขายรายวัน | ทุก tier |
| **จัดการสินค้า** | เพิ่ม/แก้ไขสินค้า | Enterprise |
| **คลังสินค้า** | รายงานสต็อกสินค้า | Enterprise |
| **ตู้จ่ายและมือจ่าย** | ตั้งตู้จ่าย หัวจ่าย ผูกถัง | ทุก tier |
| **โปรโมชั่น** | ตั้งโปรแถม/ส่วนลด | Pro, Enterprise |
| **ราคาน้ำมัน** | แก้ราคาขายต่อลิตร | ทุก tier |
| **เครื่องพิมพ์** | จับคู่ Bluetooth, ทดสอบพิมพ์ | ทุก tier |
| **สำรองข้อมูล** | Backup / Restore / Cloud* | Local ทุก tier, Cloud Pro+ |
| **ผู้ใช้งาน** | จัดการ account | ทุก tier |
| **บันทึกกิจกรรม** | Audit log | ทุก tier |
| **ตั้งค่าทั่วไป** | ชื่อปั๊ม, ใบเสร็จ, License | ทุก tier |

### 11.1 ตั้งค่าเครื่องพิมพ์

1. Backend → **เครื่องพิมพ์**
2. เปิด Bluetooth บนเครื่อง
3. ค้นหาและเชื่อมต่อเครื่องพิมพ์
4. กด **ทดสอบพิมพ์** ตรวจว่าออกกระดาษถูกต้อง
5. เลือกขนาดกระดาษ **58 mm** หรือ **80 mm**

![ตั้งค่าเครื่องพิมพ์](manual/images/20-printer-settings.png)

*รูปที่ 20 — ตั้งค่าเครื่องพิมพ์ Bluetooth*

### 11.2 ตั้งราคาน้ำมัน

1. Backend → **ราคาน้ำมัน**
2. แก้ราคาต่อลิตรของแต่ละประเภท
3. บันทึก — ราคาใหม่มีผลทันทีที่หน้าขาย

![ตั้งราคาน้ำมัน](manual/images/21-fuel-price.png)

*รูปที่ 21 — ตั้งราคาน้ำมัน*

### 11.3 ออกแบบใบเสร็จ

1. Backend → **ตั้งค่าทั่วไป** → **ออกแบบหัว/ท้ายใบเสร็จ**
2. จัดเรียง block (โลโก้, ชื่อร้าน, รายละเอียด ฯลฯ)
3. บันทึกและทดสอบพิมพ์

![ออกแบบใบเสร็จ](manual/images/22-receipt-designer.png)

*รูปที่ 22 — Receipt Designer*

## 12. License และ Package

### Package ที่รองรับ

| Package | สิทธิ์หลัก |
|---------|-----------|
| **Free** | ขายน้ำมัน, ตั้งราคา, ตู้จ่าย, backup local |
| **Standard** | + ปรับยอดถัง manual |
| **Pro** | + นำเข้าน้ำมัน, คลังเต็ม, โปรโมชั่น, cloud backup |
| **Enterprise** | + ขายสินค้า, จัดการสินค้า, คลังสินค้า |

### ตรวจสอบ / อัปเดต License

1. Backend → **ตั้งค่าทั่วไป**
2. ดู **Product Key**, **Package**, วันหมดอายุ
3. กด **ตรวจสอบ License ใหม่** เมื่อ Package เปลี่ยนบนเซิร์ฟเวอร์

![License ในตั้งค่าทั่วไป](manual/images/23-license.png)

*รูปที่ 23 — ข้อมูล License*

### Offline Grace

- หากไม่มีอินเทอร์เน็ต แอปใช้ License ที่ cache ไว้ได้ **7 วัน** หลังยืนยันล่าสุด
- หน้าตั้งค่าทั่วไปจะแสดงสถานะ grace / grace หมด

## 13. สำรองและกู้คืนข้อมูล

1. Backend → **สำรองข้อมูล**
2. **สำรองทันที** — บันทึกไฟล์ `.db` ในเครื่อง
3. **แชร์ / บันทึกไปที่อื่น** — ส่งออกไฟล์ backup
4. **กู้คืน** — เลือกไฟล์ backup (ระบบสำรองก่อน restore อัตโนมัติ)

![หน้าสำรองข้อมูล](manual/images/24-backup.png)

*รูปที่ 24 — สำรองและกู้คืนข้อมูล*

> **แนะนำ:** สำรองข้อมูลอย่างน้อยวันละ 1 ครั้ง หรือเปิด auto backup local

## 14. แก้ปัญหาเบื้องต้น

| อาการ | วิธีแก้ |
|-------|---------|
| เข้าระบบไม่ได้ | ตรวจชื่อผู้ใช้/รหัสผ่าน, ติดต่อ admin reset |
| ขายไม่ได้ / ไม่มีกะ | เปิดกะก่อน (Login แล้วเปิดกะ) |
| สต็อกไม่พอ | ตรวจคลังน้ำมัน / สินค้าใน Backend |
| พิมพ์ไม่ได้ | ตรวจ Bluetooth, จับคู่ใหม่, ทดสอบพิมพ์; ระบบลอง 3 ครั้งอัตโนมัติ |
| ขายสำเร็จแต่ไม่มีใบเสร็จ | ไปหน้าสำเร็จ → **ลองพิมพ์อีกครั้ง** |
| License ไม่อัปเดต | ต้องมีเน็ต → กดตรวจสอบ License ใหม่ |
| ไม่มีเน็ต | ใช้งานได้ภายใน grace 7 วัน (หลังยืนยันล่าสุด) |
| แอป error ตอนเปิด | ตรวจ DB — ลอง restore จาก backup ล่าสุด |
| Session หลุด | Login ใหม่ (timeout 30 นาทีไม่ใช้งาน) |

### ติดต่อผู้ดูแลระบบ

- ตรวจ **บันทึกกิจกรรม** (Audit log) ใน Backend
- ส่งไฟล์ log: `app_errors.log` (ในโฟลเดอร์เอกสารของแอป) หากแอป crash ซ้ำ
